# CLAUDE.md — Claude Code + DeepSeek V4 Pro 工作流优化配置

## 🧠 模型特性

- **模型**: DeepSeek V4 Pro
- **上下文窗口**: 128K tokens
- **擅长**: 代码生成、推理、中英文混合场景
- **注意**: 不支持视觉/图像输入；工具调用需严格遵循 JSON 格式

## 📐 代码风格与规范

### 通用原则
- **语言**: 优先使用 TypeScript（前端）/ Python（后端）
- **命名**: 驼峰式（camelCase）变量/函数，PascalCase 类/组件
- **注释**: 中文注释 + 英文代码，关键逻辑必须有注释
- **错误处理**: 使用 try-catch 包裹异步操作，错误信息中英文均可

### 前端规范
- 使用 React + TypeScript + Tailwind CSS
- 组件文件命名: `ComponentName.tsx`
- 状态管理: React Query（服务端状态） + Zustand（客户端状态）
- API 请求封装统一 `api.ts` 模块

### 后端规范
- 使用 FastAPI（Python）或 Next.js API Routes
- 路由命名: `/api/v1/resource`
- 数据验证: Pydantic / Zod
- 数据库: SQLAlchemy + PostgreSQL / Prisma + PostgreSQL

## 🔧 工具使用规则

### 文件操作
- 读取文件优先使用 `read_md`（仅限 `files/` 目录下的 .md 文件）
- 写入文件使用 `to_md`（仅限 `files/` 目录）
- 其他文件类型需通过代码逻辑处理

### 天气查询
- 城市名支持中英文
- 返回格式: `城市 + 天气状况 + 温度范围`
- 未知城市返回错误提示

## 💬 对话策略

### 多轮对话上下文维护
- 用户未指定城市时，默认沿用上一轮对话的城市
- 用户说"它/那里/这边"等代词时，自动关联最近提到的实体
- 每轮回复控制在 3-5 个要点内，避免信息过载

### 回答风格
- **技术问题**: 先给结论，再给代码示例，最后解释原理
- **非技术问题**: 简洁直接，可适当用 emoji 增加亲和力
- **错误场景**: 先道歉，再给出解决方案，最后预防建议

## ⚡ 性能优化建议

1. **DeepSeek V4 Pro 特性利用**: 利用其强大的推理能力处理复杂逻辑，简单任务可减少 token 消耗
2. **上下文管理**: 定期总结中间结果，避免上下文过长导致遗忘
3. **工具调用**: 一次调用多个独立工具（并行），减少往返延迟
4. **代码复用**: 提取公共逻辑为工具函数，减少重复生成
